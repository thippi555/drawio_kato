# drawio_kato RAG実装設計書

## プロジェクト概要
- **プロジェクト名**: drawio_kato
- **リージョン**: ap-northeast-1
- **ランタイム**: python3.12
- **IaC**: Terraform

## システム概要

Bedrockを利用したRAG（Retrieval-Augmented Generation）基本構成です。ユーザーのクエリに対して、S3に保存されたドキュメントから関連情報を検索し、Claudeで回答を生成します。

## アーキテクチャ

### リクエストフロー
1. ユーザーがAPI Gateway経由でクエリを送信
2. Lambda（drawio-kato-task-processor）がリクエストを受け取る
3. Step Functions（drawio-kato-task-flow）がワークフロー制御
4. ドキュメント検索フェーズ：S3（tasks/）からドキュメント取得
5. Bedrock Claude Haiku呼び出し：検索結果とクエリでプロンプト生成
6. 結果をS3（outputs/）に保存、DynamoDBに記録

### 主要コンポーネント

#### API Gateway
- エンドポイント: `POST /tasks`
- リクエスト形式: JSON（query, document_type等）
- レスポンス: task_id, status

#### Lambda関数
- **関数名**: drawio-kato-task-processor
- **役割**: リクエスト検証、Step Functions起動、結果集約
- **タイムアウト**: 60秒

#### Step Functions
- **ステートマシン名**: drawio-kato-task-flow
- **フロー**:
  1. ValidateInput: リクエスト検証
  2. SearchDocuments: S3からドキュメント検索
  3. GeneratePrompt: RAGプロンプト生成
  4. CallBedrock: Claude Haiku呼び出し
  5. SaveResults: 結果保存

#### Bedrock
- **モデルID**: global.anthropic.claude-haiku-4-5-20251001-v1:0
- **用途**: RAG応答生成
- **入力**: 検索ドキュメント + ユーザークエリ

#### S3（drawio-kato-artifacts）
- **tasks/**: 入力ドキュメント、クエリ
- **outputs/**: 生成結果（design.md, artifact.json）
- **prompts/**: RAGプロンプトテンプレート
- **logs/**: 処理ログ

#### DynamoDB（ai_agent_tasks）
- **主キー**: task_id (String)
- **属性**:
  - query: クエリテキスト
  - status: processing/completed/failed
  - created_at: タイムスタンプ
  - result_s3_path: 結果ファイルパス
  - document_count: 検索ドキュメント数
  - model_id: 使用モデルID

## 実装詳細

### ドキュメント検索戦略
- S3 tasks/配下のMarkdownファイルをスキャン
- ファイル名とメタデータでフィルタリング
- 最大10ファイルまで取得（コスト最適化）

### プロンプト構成
```
システムプロンプト（prompts/system.txt）:
- RAGアシスタントの役割定義
- 出力形式指定

ユーザープロンプト:
- 検索ドキュメント（コンテキスト）
- ユーザークエリ
```

### 出力ファイル
- **design.md**: 回答と根拠
- **artifact.json**: 構造化メタデータ
- **architecture.drawio**: 空（Lambda側で生成）

## セキュリティ
- IAM: Lambda→S3/DynamoDB/Bedrock最小権限
- S3: ブロックパブリックアクセス有効
- DynamoDB: 暗号化有効

## コスト最適化
- Claude Haiku使用（低コスト）
- ドキュメント数制限（最大10ファイル）
- Lambda: 512MB、60秒タイムアウト
- DynamoDB: オンデマンド課金
