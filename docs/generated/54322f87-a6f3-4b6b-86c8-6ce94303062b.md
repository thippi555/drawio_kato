# drawio_kato - Lambda + Bedrock + S3 構成設計書

## プロジェクト概要

**プロジェクト名**: drawio_kato  
**リポジトリ**: https://github.com/thippi555/drawio_kato  
**AWSリージョン**: ap-northeast-1  
**ランタイム**: Python 3.12  
**IaC**: Terraform  

## システムアーキテクチャ

### 全体フロー

```
User → API Gateway → Lambda → Step Functions → Bedrock → S3/DynamoDB
```

## AWS サービス構成

### 1. API Gateway
- **エンドポイント**: POST /tasks
- **役割**: ユーザーリクエストの受け付け
- **認証**: なし（個人利用）

### 2. Lambda
- **関数名**: drawio-kato-task-processor
- **ランタイム**: Python 3.12
- **役割**: リクエスト処理、Step Functions実行、レスポンス返却
- **タイムアウト**: 60秒
- **メモリ**: 256MB

### 3. Step Functions
- **ステートマシン名**: drawio-kato-task-flow
- **役割**: タスク処理フロー制御
- **ステップ**:
  1. タスク検証
  2. Bedrock呼び出し
  3. 結果処理
  4. S3保存
  5. DynamoDB記録

### 4. Bedrock
- **モデルID**: global.anthropic.claude-haiku-4-5-20251001-v1:0
- **役割**: AI推論（設計書生成、図解生成）
- **入力**: プロンプト、コンテキスト
- **出力**: テキスト、JSON構造化データ

### 5. Amazon S3
- **バケット名**: drawio-kato-artifacts
- **ストレージプレフィックス**:
  - `tasks/`: 入力タスク定義
  - `outputs/`: 生成結果
  - `prompts/`: プロンプトテンプレート
  - `logs/`: 処理ログ
- **出力ファイル**:
  - `design.md`: 設計書Markdown
  - `architecture.drawio`: draw.io XML形式図
  - `artifact.json`: 構造化JSON

### 6. DynamoDB
- **テーブル名**: ai_agent_tasks
- **パーティションキー**: task_id (String)
- **ソートキー**: created_at (Number)
- **属性**:
  - task_id: タスク識別子
  - created_at: 作成タイムスタンプ
  - status: 処理ステータス (pending/processing/completed/failed)
  - input_data: 入力データ
  - output_s3_path: S3出力パス
  - model_id: 使用モデルID
  - execution_time_ms: 実行時間
  - error_message: エラー情報（失敗時）

## データフロー

### リクエスト処理フロー

1. **ユーザーリクエスト**
   - API Gateway POST /tasks
   - ペイロード: JSON形式のタスク定義

2. **Lambda処理**
   - リクエスト検証
   - DynamoDBにタスク記録（status: pending）
   - Step Functions実行開始
   - task_idをユーザーに返却

3. **Step Functions実行**
   - タスク検証ステップ
   - Bedrock呼び出しステップ
   - 結果処理ステップ
   - S3保存ステップ
   - DynamoDB更新ステップ

4. **Bedrock推論**
   - プロンプトテンプレート取得（S3 prompts/）
   - ユーザー入力を組み込み
   - Claude Haiku呼び出し
   - 結果取得

5. **結果保存**
   - S3 outputs/に3ファイル保存
   - DynamoDBにステータス更新（status: completed）
   - ログをS3 logs/に記録

## セキュリティ考慮事項

- Lambda実行ロール: Bedrock、S3、DynamoDB、CloudWatch Logsへのアクセス権限
- S3バケット: 暗号化有効化（SSE-S3）
- DynamoDB: 暗号化有効化
- API Gateway: リクエスト検証、レート制限（個人利用のため軽量）

## コスト最適化

- Lambda: オンデマンド課金（低トラフィック想定）
- Bedrock: 使用量ベース課金
- S3: 標準ストレージ（アーカイブ不要）
- DynamoDB: オンデマンド課金モード
- Step Functions: 実行回数ベース課金
