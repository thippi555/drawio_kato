# drawio_kato AWS システム構成設計書

## プロジェクト概要

**プロジェクト名**: drawio_kato  
**リポジトリ**: https://github.com/thippi555/drawio_kato  
**AWS リージョン**: ap-northeast-1  
**ランタイム**: Python 3.12  
**IaC**: Terraform  

## システムアーキテクチャ

### 全体フロー

```
User
  ↓
API Gateway (POST /tasks)
  ↓
Lambda (drawio-kato-task-processor)
  ↓
Step Functions (drawio-kato-task-flow)
  ↓
Bedrock (claude-haiku-4-5-20251001-v1:0)
  ↓
S3 (drawio-kato-artifacts) + DynamoDB (ai_agent_tasks)
```

## AWS サービス構成

### 1. API Gateway
- **エンドポイント**: POST /tasks
- **役割**: ユーザーリクエストの受け入れ
- **統合先**: Lambda (drawio-kato-task-processor)

### 2. AWS Lambda
- **関数名**: drawio-kato-task-processor
- **ランタイム**: Python 3.12
- **役割**: リクエスト処理、Step Functions 起動
- **タイムアウト**: 適切に設定

### 3. AWS Step Functions
- **ステートマシン名**: drawio-kato-task-flow
- **役割**: ワークフロー制御、Bedrock 呼び出し調整
- **状態遷移**: タスク処理 → Bedrock 実行 → 結果保存

### 4. Amazon Bedrock
- **モデル ID**: global.anthropic.claude-haiku-4-5-20251001-v1:0
- **役割**: AI による設計書・アーキテクチャ生成
- **入力**: タスク定義、プロンプト
- **出力**: 構造化テキスト、JSON

### 5. Amazon S3
- **バケット名**: drawio-kato-artifacts
- **ストレージプレフィックス**:
  - `tasks/`: 入力タスク定義
  - `outputs/`: 生成結果
  - `prompts/`: プロンプトテンプレート
  - `logs/`: 実行ログ
- **出力ファイル**:
  - `design.md`: Markdown 形式設計書
  - `architecture.drawio`: Draw.io XML
  - `artifact.json`: 構造化 JSON

### 6. Amazon DynamoDB
- **テーブル名**: ai_agent_tasks
- **役割**: タスク履歴、メタデータ管理
- **主キー**: task_id
- **属性**: timestamp, status, input, output, model_id

## データフロー

1. **リクエスト受信**: ユーザーが API Gateway に POST
2. **タスク登録**: Lambda が DynamoDB にタスク記録
3. **ワークフロー起動**: Step Functions 実行開始
4. **AI 処理**: Bedrock に設計生成を依頼
5. **結果保存**: S3 に design.md, architecture.drawio, artifact.json を保存
6. **メタデータ更新**: DynamoDB にステータス更新
7. **レスポンス返却**: ユーザーに結果 URL を返却

## ストレージ構成

### S3 ディレクトリ構造
```
drawio-kato-artifacts/
├── tasks/
│   └── {task_id}/input.json
├── outputs/
│   └── {task_id}/
│       ├── design.md
│       ├── architecture.drawio
│       └── artifact.json
├── prompts/
│   ├── system_prompt.txt
│   └── design_template.txt
└── logs/
    └── {task_id}/execution.log
```

### DynamoDB スキーマ

**テーブル**: ai_agent_tasks

| 属性名 | 型 | 説明 |
|--------|-----|------|
| task_id | String (PK) | タスク一意識別子 |
| timestamp | Number | 作成タイムスタンプ |
| status | String | PENDING / PROCESSING / COMPLETED / FAILED |
| input_s3_path | String | 入力ファイル S3 パス |
| output_s3_path | String | 出力ファイル S3 パス |
| model_id | String | 使用 Bedrock モデル |
| user_id | String | ユーザー識別子 |
| error_message | String | エラー時のメッセージ |

## 出力ファイル仕様

### design.md
- Markdown 形式の設計書
- AWS サービス構成、フロー、仕様を記載
- GitHub で直接表示可能

### architecture.drawio
- Draw.io XML 形式
- Lambda 側で artifact.json から自動生成
- ビジュアルアーキテクチャ図

### artifact.json
- 構造化 JSON
- system, services, workflow, storage, dynamodb, files, future_extensions を含む
- プログラマティック処理用

## セキュリティ・コスト最適化

- **個人利用**: IAM ロール最小権限設定
- **低コスト**: Haiku モデル使用、オンデマンド課金
- **サーバレス**: 管理サーバ不要
- **ログ管理**: CloudWatch Logs で監視
