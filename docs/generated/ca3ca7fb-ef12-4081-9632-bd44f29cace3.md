# drawio_kato - Lambda + Bedrock + S3 構成設計書

## プロジェクト概要

- **プロジェクト名**: drawio_kato
- **リポジトリ**: https://github.com/thippi555/drawio_kato
- **AWSリージョン**: ap-northeast-1
- **ランタイム**: Python 3.12
- **IaC**: Terraform

## システムアーキテクチャ

### 全体フロー

```
User
  ↓
API Gateway (POST /tasks)
  ↓
Lambda (drawio-kato-task-processor)
  ↓
Step Functions (drawio-kato-task-flow)
  ↓
Bedrock (claude-haiku-4-5-20251001-v1:0)
  ↓
S3 (drawio-kato-artifacts)
  ↓
DynamoDB (ai_agent_tasks)
```

## AWSサービス構成

### API Gateway
- **エンドポイント**: POST /tasks
- **役割**: ユーザーリクエストの受け入れ
- **統合先**: Lambda (drawio-kato-task-processor)

### Lambda
- **関数名**: drawio-kato-task-processor
- **ランタイム**: python3.12
- **役割**: リクエスト処理、Step Functions実行
- **タイムアウト**: 60秒

### Step Functions
- **ステートマシン名**: drawio-kato-task-flow
- **役割**: タスク処理フロー制御
- **処理内容**:
  1. タスク検証
  2. Bedrock呼び出し
  3. 結果処理
  4. S3保存
  5. DynamoDB更新

### Bedrock
- **モデルID**: global.anthropic.claude-haiku-4-5-20251001-v1:0
- **役割**: AI処理（設計書生成、アーキテクチャ分析）
- **入力**: プロンプト（S3から取得）
- **出力**: 構造化テキスト

### S3
- **バケット名**: drawio-kato-artifacts
- **ストレージプレフィックス**:
  - `tasks/`: 入力タスク定義
  - `outputs/`: 出力ファイル
  - `prompts/`: Bedrockプロンプトテンプレート
  - `logs/`: 処理ログ
- **出力ファイル**:
  - `design.md`: 設計書Markdown
  - `architecture.drawio`: Draw.io XML形式
  - `artifact.json`: 構造化JSON

### DynamoDB
- **テーブル名**: ai_agent_tasks
- **パーティションキー**: task_id (String)
- **ソートキー**: timestamp (Number)
- **属性**:
  - task_id: タスク識別子
  - timestamp: 作成時刻
  - status: 処理ステータス (pending, processing, completed, failed)
  - input_data: 入力データ
  - output_location: S3出力パス
  - created_at: 作成日時
  - updated_at: 更新日時

## データフロー

### リクエスト処理フロー

1. **入力**: ユーザーがAPI Gatewayに POST /tasks でリクエスト送信
2. **Lambda処理**: 
   - リクエスト検証
   - タスクID生成
   - DynamoDBに初期レコード作成 (status: pending)
   - Step Functions実行開始
3. **Step Functions処理**:
   - タスク情報取得
   - Bedrockへプロンプト送信
   - 結果取得
4. **Bedrock処理**:
   - プロンプトテンプレート適用
   - AI処理実行
   - 結果生成
5. **出力処理**:
   - design.md 生成
   - artifact.json 生成
   - architecture.drawio 生成
   - S3に保存 (outputs/ プレフィックス)
6. **完了処理**:
   - DynamoDBレコード更新 (status: completed, output_location)
   - ユーザーに結果URL返却

## セキュリティ考慮事項

- Lambda実行ロールに最小権限ポリシー適用
- S3バケットは暗号化有効化
- DynamoDBはオンデマンド課金モデル
- Bedrockアクセスは IAM ロールベース

## コスト最適化

- Lambda: 無料枠内での運用想定
- DynamoDB: オンデマンド課金（低トラフィック対応）
- S3: ライフサイクルポリシーで古いログ削除
- Bedrock: 使用量ベース課金
