# drawio_kato RAG基本構成設計書

## プロジェクト概要

本設計書は、AWS Bedrockを利用したRAG（Retrieval-Augmented Generation）基本構成をdrawio_katoプロジェクトに統合するための設計です。

## システムアーキテクチャ

### 全体フロー

1. ユーザーがAPI Gateway経由でクエリを送信
2. Lambda関数がクエリを受け取り、Step Functionsを起動
3. Step Functionsが以下の処理を順序実行
   - クエリの埋め込み生成（Bedrock Embeddings）
   - DynamoDBからの類似ドキュメント検索
   - 検索結果とクエリをプロンプトに統合
   - Bedrock Claude Haikuで回答生成
4. 結果をS3に保存し、DynamoDBにメタデータを記録

## AWS サービス構成

### API Gateway
- エンドポイント: POST /tasks
- リクエスト: クエリテキスト、オプションで検索パラメータ
- レスポンス: タスクID、処理ステータス

### Lambda関数
- 関数名: drawio-kato-task-processor
- ランタイム: python3.12
- 責務:
  - リクエスト検証
  - Step Functions実行トリガー
  - エラーハンドリング

### Step Functions (drawio-kato-task-flow)

#### ステート定義

1. **ValidateQuery**: クエリの妥当性チェック
2. **GenerateEmbedding**: Bedrock Embeddings APIでクエリを埋め込みベクトル化
3. **SearchDocuments**: DynamoDBクエリでベクトル類似度検索（スキャン+フィルタ）
4. **BuildPrompt**: 検索結果をコンテキストとしてプロンプト構築
5. **InvokeBedrockModel**: Claude Haikuで回答生成
6. **SaveResults**: 結果をS3に保存
7. **UpdateMetadata**: DynamoDBにタスク完了メタデータを記録

### Bedrock

#### モデル
- モデルID: global.anthropic.claude-haiku-4-5-20251001-v1:0
- 用途: 回答生成

#### Embeddings
- 用途: クエリとドキュメントのベクトル化
- 推奨: Amazon Titan Embeddings v2

### DynamoDB

#### テーブル: ai_agent_tasks

**パーティションキー**: task_id (String)
**ソートキー**: created_at (Number, Unix timestamp)

**属性**:
- task_id: タスク一意識別子
- created_at: 作成タイムスタンプ
- query: ユーザークエリテキスト
- query_embedding: クエリのベクトル表現（リスト）
- status: 処理ステータス (pending, processing, completed, failed)
- retrieved_docs: 検索結果ドキュメントID配列
- response: 生成された回答テキスト
- s3_output_path: S3保存パス
- execution_time_ms: 処理時間
- error_message: エラー時のメッセージ
- updated_at: 更新タイムスタンプ

#### テーブル: rag_documents (将来拡張)

**パーティションキー**: doc_id (String)
**ソートキー**: version (Number)

**属性**:
- doc_id: ドキュメント一意識別子
- version: ドキュメントバージョン
- content: ドキュメント本文
- embedding: ドキュメントのベクトル表現（リスト）
- metadata: ドキュメントメタデータ（JSON）
- created_at: 作成タイムスタンプ

### S3 (drawio-kato-artifacts)

**ストレージプレフィックス**:
- tasks/: タスク定義ファイル
- outputs/: 処理結果ファイル
- prompts/: プロンプトテンプレート
- logs/: 処理ログ

**出力ファイル**:
- design.md: 設計ドキュメント
- architecture.drawio: アーキテクチャ図
- artifact.json: 構造化メタデータ

## セキュリティ考慮事項

- IAM ロール: Lambda、Step Functions、Bedrock、S3、DynamoDBへの最小権限アクセス
- 暗号化: S3サーバサイド暗号化、DynamoDB暗号化
- ログ: CloudWatch Logsへの処理ログ記録

## コスト最適化

- Bedrock: 従量課金（トークン単位）
- Lambda: 実行時間ベース（python3.12）
- DynamoDB: オンデマンド課金モード推奨
- S3: ライフサイクルポリシーで古いログを削除
